Football LiveScores App

Why

After obtaining a Software Development role in the NHS, I had to learn a lot to improve my skills with C# and Object Oriented Programming. The team I am in creates desktop applications using WPF and the MVVM pattern. So in order to learn and improve my skills I decided to start work on this project as I am a big football fan and was very interested in using the Live Score API which contains football data from multiple countries and competitions, including past and future matches, league tables, live scores, events and statistics.

Problems

As this was one of my first times using WPF and xaml it took me some time to understand what could be done with it. And from the start I had a few problems, especially when trying to display data through data bindings.

  • The first was just loading in the list of countries from the API and trying to display them in a Data Grid.

  • I had been using a modified Messenger class taken from MVVM Light that is used send data between ViewModels. There was a problem where I was receiving a number that was different to what I was expecting which meant the wrong tab was being displayed.

  • At one point the API got updated so that the leagues part that I was using got deprecated. It was still able to be used, however there were certain games and league tables that were causing errors and weren't able to be displayed.

Solutions

  • Displaying just the list of countries in a Data Grid was quite difficult in the beginning. I managed to load the data from the API into a list of country objects and after a lot of trial and error and help from StackOverflow, I was able to bind that list to the Data Grid and display it when the application loaded.

  • So this took me a while to figure out what was wrong as it was my first time using a Messenger class and I didn't fully understand how it worked. Anyway in the end it was quite a simple solution, as when triggering the event to change the tab index it was also triggering another event that was also sending a different number. This meant that the tab index was changing to the last number it received. This was fixed by sending a string instead of a number with more info about where it has come from, this way when receiving the string I was able to handle it and find out exactly what to do with each message.

  • For this problem I had to refactor a lot of my code to account for the updated API which also gave me the chance to make it a little more efficient.

As with any project there were many more problems and little bugs that arose. With some debugging and help from my good friend Google hopefully those that arose are now fixed. And for those that are still yet to arise with more use and testing they will be handled and logged so they can be investigated further.

What I Learnt

From working on this Project I have learnt a massive amount about C#, OOP the MVVM pattern and how important it is to keep your code loosely coupled so that it may be reused and easier to test. I am still working on this in my spare time and adding little features here and there, if you would like to use it for yourself you can download the folder from my GitHub here, extract the Zipped file and run the .exe file in the \bin\Debug folder.

Why

This was created when I was learning React as part of a Udemy course that I was following. I enrolled in this course after University as a way to maintain my html and css skills while also going more in depth with JavaScript and some of it's frameworks (React). This was the example project that I followed throughout the course. I also attempted creating my own application which can be found here. Project Portfolio

Why

About 6-12 months after University and into my job search I decided to create an updated and more professional looking Portfolio with some of the new skills that I had learnt. I wanted to display all of the projects that I had created over the years on a web page that I was proud of and to challenge myself to designing a modern and simplistic UI. Just incase you weren't aware you are currently on this web page and any new projects and applications I create will be uploaded to this site.

Problems

When it comes to Web Design I feel I still have a lot to learn about css. I don't find it quite as easy or as logical as html or JavaScript. As a result designing the UI of the web page took me longer than expected and came with lot's of little problems when trying to display everything exactly where I wanted it to be displayed.
The only other significant problem I had was when I was creating the contact form and trying to get it to send the message to my email address.

Solutions

As for the css and the UI I had to do a lot of Googling and trial and error, the W3 Schools website was a very useful tool and a big help. I also found testing after making frequent small changes was much better than making one big change and finding something was broken, it was a lot easier to determine the root cause of the problem when only making a small change, as I find css to be quite temperamental.
As for the contact form I managed to use some PHP that I found on StackOverflow and modified, I also included some validation to prevent users from entering redundant or harmful data. I then customized the look and feel of the form so that it was all contained within the web page and matched the rest of the UI design.

What I Learnt

As I progressed through this project I learnt more and more about css and the things that you can do with just css alone such as animations and transitions. I also learnt more about JavaScript and in particular ES6 and some it's new features and syntax. And I learnt how to create a fully functional and secure contact form with the use of PHP, jQuery and AJAX.

iOS Game

Why

This iOS application was created as one of my final University projects. We were given the base code for a normal game of noughts and crosses and were tasked with modifying the game and adding more features. I did this by making the grid bigger so the players had to get 4 in a row rather than 3, adding a 10 second timer so if you run out of time you miss your go and also there was a quiz in which you have to answer a question correctly in order to take your turn on the board.

Problems

As I am writing this a couple of years after completing, I can't recall all the major problems that occurred. However as with any project there are many problems and bugs that will arise especially as this was the first time I had used swift and XCode.

Solutions

When using a new technology there are times that you need to call a friend (Google) in order to find out how something is done. Google was used a lot throughout this project in order to understand how swift handles things differently compared to other programming languages. And as for any little bugs I like step through my code and debug line by line to see where it may be going wrong before resorting to my call a friend life line.

What I Learnt

I learned a lot about mobile application development. We started off learning a little bit of Objective-C, before moving onto swift which is the language that was used for this application.
